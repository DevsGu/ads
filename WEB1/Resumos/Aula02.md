&#x20;   1. Como funciona o processo de compilação do Java?

&#x20;   2. O que é um objeto no Java?



R1:. O compilador do Java transforma o código fonte .java em bytecodes. Esses bytecodes são interpretados pela JVM (máquina virtual do Java) e assim é possível que a JVM consiga rodar em qualquer tipo de sistema operacional.



R2:. Objeto no Java é a instância da classe. Quero dizer, tudo que é de concreto ou abstrato pode ser visto como uma classe (molde), que depois de instanciado se torna um objeto.



&#x20;   3. Se você tivesse que criar um cardápio de uma hamburgueria, qual estrutura de decisão você usaria?

&#x20;   4. Qual a diferença do compilador para o interpretador?



R3:. Utilizaria switch case, que é amplamente utilizado para esse propósito: uma lista de opções.



R4:. O compilador traduz todo o código, depois executa. Me lembra bastante o processo de modelo de software cascata, que primeiro elabora todo o projeto para depois executá-lo. O interpretador é semelhante ao modelo de software incremental, que cria o projeto por partes. Nesse caso, o interpretador interpreta palavra por palavra e já vai executando.



&#x20;   5. Dados básicos do Java

&#x20;   a) Pacote

&#x20;   b) Importação

&#x20;   c) Classe

&#x20;   d) Objeto



R5A:. Os pacotes são como pastas que utilizamos para colocarmos os códigos.



R5B:. Utilizamos as importações para usar as classes que já estão presentes na biblioteca do Java.



R5C:. Classe é o molde do objeto, pode ser algo concreto ou abstrato.



R5D:. Objeto é qualquer coisa que seja concreta ou abstrata instanciada pela classe.



&#x20;   6. Explique o que é encapsulamento



R6:. Utilizamos o encapsulamento nos atributos, métodos e classes para que nem todos possam ter acesso aos mesmos. Sendo assim, possuímos os métodos de acesso public, private e protected. Para acessar métodos privados, utilizamos os métodos get e set. get busca dados, set pode modificar.
